# TeamPilot 使用说明书

## 1. 登录与角色

打开前端地址后使用账号密码登录。默认本地开发账号：

- 管理员：`admin / admin123`
- 演示成员：`zhangsan / 123456`、`lisi / 123456`、`wangwu / 123456`

系统按角色和权限控制按钮显示。管理员可以在系统设置中调整角色权限。

## 2. 项目管理

项目管理页是日常使用主入口。

- 顶部汇总区展示项目数、进行中项目、任务总数、已完成任务和完成率。
- “进展更新”用于粘贴群消息，由 AI 识别任务进展，确认后写入任务历史。
- “每日巡检报告”用于查看日报或周报，并支持邮件发送。
- “新建项目”同时支持手动填写和 AI 生成项目计划。

项目行可展开任务树。项目状态由任务自动推导：

- `规划中`：没有任务，或任务都还没到开始时间。
- `进行中`：存在已开始且未完成的任务。
- `已完成`：所有未删除任务都已会签完成。
- `已归档`：管理员手动归档。

## 3. 任务管理

任务支持父子层级、多人负责人、工时、开始日期、截止日期、进度和会签。

- 点击任务名称打开任务详情抽屉，可编辑任务名称、目标和描述。
- 点击“查看记录”打开进展抽屉，可手动提交真实进展。
- 负责人支持多选，列表中会完整显示所有负责人。
- 有子任务的任务可以折叠，子任务支持同级拖拽排序。
- 删除任务为软删除，子任务会同步进入删除状态；恢复父任务后才能恢复子任务。

任务状态只保留三种：

- `待开始`：开始日期晚于当前时间。
- `进行中`：已到开始日期且未完成。
- `已完成`：进度达到 100% 后，由有权限角色会签确认。

## 4. 进展更新

进展更新是任务状态和进度变更的主入口。

粘贴格式建议：

```text
姓名
时间
任务进展
```

一个人的多条任务进展可以写在同一段。AI 识别后会给出匹配项目、匹配任务、进度百分比、说明和置信度。用户确认后，系统会写入任务进展历史。

进度达到 100% 不会自动完成任务，仍需会签确认。

## 5. 每日巡检报告与周报

项目管理页的“每日巡检报告”支持两种报告：

- 日报：优先调用 AI，输出今日风险项目、逾期/阻塞任务、未更新成员、优先推进任务和待会签任务。
- 周报：按最近 7 天的进展、会签、逾期和优先级生成。

报告可以通过邮件发送。部署前需要配置 SMTP：

```env
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USERNAME=robot@example.com
SMTP_PASSWORD=your-password
SMTP_FROM_EMAIL=teampilot@example.com
SMTP_USE_TLS=true
SMTP_USE_SSL=false
REPORT_DEFAULT_RECIPIENTS=pm@example.com,lead@example.com
```

如果页面里填写了接收人，优先使用页面输入；否则使用 `REPORT_DEFAULT_RECIPIENTS`。

## 6. 仪表盘

仪表盘用于管理视角查看整体情况。

- 概览：总任务数、进行中、已逾期、完成率。
- 我的任务：按紧急/重要四象限展示当前登录人的未完成任务。
- 项目进度概览：展示项目完成度和风险标签。
- 团队工作负荷：按任务负责人统计未完成工作量。

团队工作负荷优先使用任务 `estimated_hours`。如果任务没有预估工时，系统按层级折算：

- 一级任务：8 小时
- 二级任务：4 小时
- 更深层子任务：2 小时

## 7. 人员管理

人员管理页可以查看成员列表、角色、部门、个人介绍和能力信息。个人介绍会进入 AI 分析上下文，用于任务推荐、估时和能力分析。

## 8. 系统设置

系统设置包含：

- 权限管理：配置角色可用功能。
- AI 配置：配置大模型接口、模型名、密钥和 prompt。
- 数据连接器：配置公司内部平台 API，供任务数据 Skill 调用。

## 9. 任务数据 Skill

任务数据 Skill 用于把内部平台数据接入任务进展。

流程：

1. 在任务详情里描述数据来源，例如“这个任务的数据从测试平台获取。接口是 GET /api/test/feature/{feature_id}/summary。”
2. 系统生成 Skill 草稿。
3. 用户测试执行并确认。
4. 后续可执行 Skill 获取外部数据。
5. 用户确认采纳后才写入任务进展历史。

AI 只能调用已配置的数据连接器，不能自由访问任意地址。

## 10. 推荐日常流程

1. 新建项目，手动或 AI 生成任务树。
2. 给任务分配负责人、工时和截止日期。
3. 每天粘贴群消息到“进展更新”。
4. 查看每日巡检报告，处理风险、逾期和待会签项。
5. 每周生成周报并邮件发送。
